前言

折腾了hexo、hugo、gridea几种静态博客生成工具，最后还是决定用hugo了。hexo依赖nodejs，而且总感觉生成速度有点慢。gridea的gui倒是挺易用，门槛也比较低，但是我不是很喜欢这样把所有细节都封装起来，因为很多东西不能自己把控，出点小问题解决起来很麻烦。hugo与前两者相比很简洁，就一个exe程序。而且还支持实时修改预览，深得我心~

另外也是抱着不再折腾的心态，主题没有弄的花里胡哨的。国人写的LoveIt感觉不错，简约而不简单。看上去很清爽，但该有的功能都有。所以目前暂定就是hugo + LoveIt的组合啦。

搭建

首先是hugo的下载，前面也说过了，hugo就一个exe程序，直接下载下来在命令行里执行相应命令就行了。我是把它加到了环境变量中，用起来更方便一点。

接着在想要创建博客的目录下执行hugo new site或者是hugo new site <path>后者可以在命令中指定生成博客的路径。这样就在对应的目录下生成了初始站点。

然后安装主题，到github下载LoveIt放在themes目录下，或者在themes目录下git clone。同时修改根目录下的config.toml文件。我暂时使用了LoveIt提供的一个最基本的配置文件，内容如下：

baseURL = "http://example.org/"
# [en, zh-cn, fr, ...] 设置默认的语言
defaultContentLanguage = "zh-cn"
# 网站语言, 仅在这里 CN 大写
languageCode = "zh-CN"
# 是否包括中日韩文字
hasCJKLanguage = true
# 网站标题
title = "CK's Blog"

# 更改使用 Hugo 构建网站时使用的默认主题
theme = "LoveIt"

[params]
  # LoveIt 主题版本
  version = "0.2.X"

[menu]
  [[menu.main]]
    identifier = "posts"
    # 你可以在名称 (允许 HTML 格式) 之前添加其他信息, 例如图标
    pre = ""
    # 你可以在名称 (允许 HTML 格式) 之后添加其他信息, 例如图标
    post = ""
    name = "文章"
    url = "/posts/"
    # 当你将鼠标悬停在此菜单链接上时, 将显示的标题
    title = ""
    weight = 1
  [[menu.main]]
    identifier = "tags"
    pre = ""
    post = ""
    name = "标签"
    url = "/tags/"
    title = ""
    weight = 2
  [[menu.main]]
    identifier = "categories"
    pre = ""
    post = ""
    name = "分类"
    url = "/categories/"
    title = ""
    weight = 3

# Hugo 解析文档的配置
[markup]
  # 语法高亮设置 (https://gohugo.io/content-management/syntax-highlighting)
  [markup.highlight]
    # false 是必要的设置 (https://github.com/dillonzq/LoveIt/issues/158)
    noClasses = false

下面新建一篇文章来看看效果吧！LoveIt的博文对应在content/posts目录下，所以我们要这样新建：hugo new posts/helloworld.md。执行这句命令以后，就创建了一篇新博文，以md文件的形式存在。写博文的话就是编辑这个md文件啦，推荐使用Typora这个markdown编辑器。要注意的一点是，每个生成的博文开头有一些属性信息：

---
title: "Blog搭建小记"
date: 2021-09-03T01:04:10+08:00
categories: ["test"]
tags: ["test"]
draft: false
---

其中draft的意思是草稿，如果设置为true的话这篇博文将不会被渲染。你可以把它设置成false或者干脆删除这行。categories和tags分别是分类和标签，没有的话需要手动添加。注意要使用类似python列表的方式，哪怕只属于一个分类。

写好博文之后就可以预览效果啦，执行hugo serve或者hugo server就可以启动hugo自带的server，然后用浏览器访问http://localhost:1313/就可以看到效果了。注意此时如果修改了文件内容并保存的话，会被实时渲染展示的哦。

发布

在本地预览没有什么问题以后，就可以发布到github上了。首先是生成静态网页文件。执行hugo --theme=LoveIt --baseUrl="http://cknightx.github.io"（如果在config.toml中已经指定了theme和baseUrl这里可以不用写，直接执行hugo即可）。这时会生成一个public文件夹，文件夹中就是刚刚渲染出的网页文件，然后将其push到github上：

cd public
git init
git remote add origin git@github.com:CknightX/cknightx.github.io.git
git add -A
git commit -m "first commit"
git push -u origin master

插入图片

markdown的图片插入语法是：![test](/static/test.png)。注意根目录下有一个static文件夹，只需要把图片放在这个文件夹下，然后使用相对路径来引用图片就可以了。
